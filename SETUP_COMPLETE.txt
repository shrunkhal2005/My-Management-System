❓ FREQUENTLY ASKED QUESTIONS:

Q: Why do I need to download the MySQL driver separately?
A: Maven attempted to download it but failed. Manually downloading ensures we get it.

Q: Can I use MariaDB instead of MySQL JDBC driver?
A: Yes! Run: Download-MySQLDriver.ps1 -UseMariaDB

Q: What happens if I run HMS without the driver?
A: It runs in OFFLINE MODE - data stored in memory, lost on close. UI still works.

Q: How do I know if it's in ONLINE mode?
A: Check Admin Panel - it will say "✅ MySQL Connected" (ONLINE) or "⚠️ OFFLINE MODE"

Q: Can I test without installing the driver?
A: Yes! Run in OFFLINE MODE. UI works perfectly, just no data persistence.

Q: Where does my data go if running offline?
A: In application memory (Java ArrayList). Lost when you close HMS.

Q: How do I switch back to OFFLINE if driver breaks?
A: Just delete the JAR file or run without -cp flag. HMS automatically detects.

🆘 IF SOMETHING GOES WRONG:

Problem: "ClassNotFoundException: com.mysql.cj.jdbc.Driver"
├─ Cause: MySQL driver not found
└─ Fix: Run Download-MySQLDriver.ps1

Problem: "Connection refused: java.sql.SQLException: Communications link failure"
├─ Cause: MySQL Server not running
└─ Fix: Start MySQL: Services → MySQL Server 9.7 → Right-click → Start

Problem: "Access denied for user 'root'@'localhost'"
├─ Cause: Wrong password
└─ Fix: Verify password in DatabaseConnection.java (line 8)

Problem: "table 'patients' doesn't exist"
├─ Cause: Database not initialized
└─ Fix: Run: setup.sql manually or delete jfx_db and let HMS recreate it

Problem: Driver downloaded but HMS still shows OFFLINE
├─ Cause: JAR in wrong location
└─ Fix: Ensure file is in d:\jfx\lib\mysql-connector-java-8.0.33.jar
